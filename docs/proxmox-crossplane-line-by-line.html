.bin/gpu-switch.sh (block-level)

blkStepWhy we pay for it
1arg → sets[] (talos|t / windows|w)Picks which VM gets the GPU (started=true) and which is stopped, incl. windows passthrough+vga=null for production.
2render() helm template --set …Produces the two EnvironmentVMs with the right power/passthrough values — no hand-editing.
3--apply: stop-VM first → sleep 60 → start-VMReview fix — sequences the GPU release before the acquire, so the passthrough start doesn't fail on a busy device.
4else: dry-run printDefault is preview-only — you must pass --apply to actually switch.
5verify hint (check Synced)Tells the operator to confirm the start didn't error on a still-busy GPU.

Justification basis: every line maps to a Proxmox .conf field, a Talos/k8s requirement, a provider constraint (1-cdrom, ISO-must-exist, import-by-VMID), a secret-handling rule (nothing secret in the public repo), or a specific review finding it fixes. Lines that are pure YAML/Helm scaffolding (apiVersion/kind/spec wrappers, range/if/end) are structural and unpriced.
