## Problem

PXE provisioning was unreliable and not AI-drivable: wrong interface/next-server selection (advertised the router IP 192.168.68.1), hidden `sudo dnsmasq` password prompt that hung, a 10-min serve-timeout that killed the server mid-install, no observability (agent relied on unreadable cmux panes + phone photos), a config-fetch detector that an operator `curl` falsely tripped, and a never-settling install loop on installed nodes.

## Decisions

- Skip Linux `CAP_NET_BIND_SERVICE` rootless path; standardize on sudo/sudoers everywhere (`CAP_NET_BIND_SERVICE` alone is insufficient — dnsmasq DHCP also needs `CAP_NET_ADMIN`+`CAP_NET_RAW`; `setcap` mutates a shared binary; nostos is macOS-operator-centric). Recorded in bead `.10`.
- Subagents run SEQUENTIALLY when they share `serve.go`/`commands.go`/`schema.go`/package `pxe` (concurrent edits corrupt each other's build/test). Only genuinely disjoint work parallelized (`.1` multi-iface, `.9` docs, `.10` decision ran together up front).
- Installed-state lives in package `pxe` (`internal/pxe/installed.go`) to stay cycle-free: serve reads it, `provisioner/pxe` (imports pxe) writes it. "Installed" trigger = config-fetch; cleared at install start in `Prepare`.
- gofmt-cleaned only files THIS session authored; left 3 pre-existing committed gofmt-dirty files (`internal/cli/cluster.go`, `dryrun/dryrun.go`, `flash_test.go`) untouched.
- Nothing committed (per safety rules); pre-existing uncommitted cilium/template files never touched.
